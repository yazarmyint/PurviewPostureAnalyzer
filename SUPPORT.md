# Support

## How to file issues and get help

Purview Posture Analyzer (PPA) uses **GitHub Issues** for bug reports and questions:
<https://github.com/yazarmyint/PurviewPostureAnalyzer/issues>

- Search the existing issues first to avoid duplicates.
- If none match, open a new issue. For a bug, include your PowerShell edition (Windows
  PowerShell 5.1 or PowerShell 7+), the command you ran, and the behavior you saw versus
  what you expected.

For a suspected **security vulnerability in the tool**, do not open a public issue - use
private reporting instead (see `SECURITY.md`).

## Scope of support

PPA is an independent, community-maintained project (see `NOTICE`) - not a Microsoft
product and not covered by Microsoft support. Help is best-effort, provided by the
maintainer and community through GitHub Issues; there is no service-level agreement.

Support covers the tool itself: installing and importing the module, running the analyzer,
and interpreting the report's five statuses. It does **not** cover remediating the
Microsoft Purview misconfigurations the tool reports in your tenant, or Microsoft 365 /
Purview licensing decisions - those are your environment's concern, and the report's
guidance is only a starting point for your own review.
